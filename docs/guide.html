Fluent Development Guide

Prerequisites

Before starting, ensure you have:

  • A ServiceNow PDI (Personal Developer Instance)
  • A GitHub account with SSH or HTTPS access configured
  • macOS or Linux (Windows WSL2 also works)
  • Node.js v20.18.0 or later — nodejs.org, choose LTS
1

Installation & Verification

1.1 — Check Existing Tools

bash 
now-sdk --version    # ServiceNow SDK (Fluent dev engine)
node --version       # Node.js — need v20.18.0+
npm --version        # npm — need v8.19.3+
git --version        # Git
gh --version         # GitHub CLI

1.2 — Install Node.js (if needed)

The ServiceNow SDK requires Node.js v20.18.0 or later.

bash 
node --version

If below v20.18.0 or not installed:

  • macOS (recommended): brew install node@20
  • All platforms: Download LTS from nodejs.org
bash — verify 
node --version   # Should show v20.18.0 or higher
npm --version    # Should show v8.19.3 or higher

1.3 — Install ServiceNow SDK (now-sdk)

What is now-sdk? The ServiceNow SDK is the Fluent pro-code development engine. It compiles Fluent DSL (TypeScript-like metadata) to ServiceNow records and manages the full app lifecycle: init, download, build, install.
bash 
npm install -g @servicenow/sdk
bash — verify 
now-sdk --version

Expected output: 4.4.x

1.4 — Install GitHub CLI (if needed)

bash 
# macOS
brew install gh

gh --version   # Verify

Authenticate:

bash 
gh auth login

Follow prompts — GitHub.com → HTTPS → authenticate via browser.

1.5 — Configure Git Identity

bash 
git config --global user.name "Your Name"
git config --global user.email "your.email@company.com"

git config --global --list | grep user  # Verify
2

Instance Authentication

The SDK has its own built-in authentication — no separate CLI tool required. Use now-sdk auth to connect to your PDI.

2.1 — Add Your Instance as an Alias

bash 
now-sdk auth --add dev

Replace dev with any short alias name you prefer. You will be prompted for:

  • Instance URL: https://your-instance.service-now.com
  • Username: Your ServiceNow login
  • Password: Your ServiceNow password
Security Credentials are stored locally by the SDK. Never hardcode them in project files or commit them to git. The .gitignore in this project already excludes credential files.

2.2 — Verify Authentication

bash 
now-sdk auth --list

Expected output shows your alias and instance URL with a connected status.

If you see an auth error: double-check the instance URL (no trailing slash), verify credentials in a browser, then re-run now-sdk auth --add dev.

2.3 — CI/CD Authentication (Optional)

For non-interactive environments (pipelines, automation), use environment variables:

bash 
export SN_SDK_INSTANCE_URL="https://your-instance.service-now.com"
export SN_SDK_USER="your-username"
export SN_SDK_USER_PWD="your-password"
export SN_SDK_NODE_ENV="SN_SDK_CI_INSTALL"
3

Create a Fluent Scoped App

What is a Fluent scoped app? In the modern SDK workflow, you write app metadata as Fluent DSL — a TypeScript-like language. The SDK compiles it to ServiceNow records and installs them to your instance. All development is local-first. If you know this already, skip to 3.1.

3.1 — Initialize the App

bash 
now-sdk init

You will be prompted to choose a template and provide:

  • App name: e.g., My Training App
  • Scope: e.g., x_snc_training
  • Version: 1.0.0
  • Instance alias: the alias from Phase 2 (e.g., dev)

3.2 — Project Structure

tree
my-training-app/
├── src/
│   ├── index.ts         # App entry point (Fluent DSL)
│   └── ...
├── now-sdk.json         # SDK project config
├── package.json
└── tsconfig.json

3.3 — Download Existing App Metadata (if applicable)

If starting from an app that already exists on your instance:

bash 
now-sdk download src/
Tip Always run now-sdk download src/ before starting work to ensure your local files are in sync with the instance.

3.4 — Verify in the Instance

Log into your PDI: App Engine Studio → My Apps (or System Applications → Studio).
Your app will appear after you run now-sdk install in Phase 5.

4

Connect to Claude Code

What is Claude Code? Claude Code (CC) is Anthropic's AI-powered CLI development environment. It understands your Fluent codebase and can help you write, review, and install ServiceNow app code — including talking directly to your instance via MCP (Phase 7).

4.1 — Open the Project in Claude Code

bash 
claude

Claude Code will start and index your project files.

4.2 — Create a CLAUDE.md

CLAUDE.md 
# My Training App — ServiceNow Fluent Scoped App

## Project Overview
ServiceNow Fluent scoped app (scope: x_snc_training) on PDI instance.
Primary development via ServiceNow SDK (now-sdk) + Claude Code.

## Key Commands
- Download from instance: `now-sdk download`
- Install to instance:    `now-sdk install`
- Build (compile only):   `now-sdk build`
- Initialize new app:     `now-sdk init`
- App source: `src/`

## Conventions
- All changes installed to instance AND committed to git
- Never commit SDK credentials or instance auth files
- Branch naming: feature/, fix/, chore/

4.3 — MCP Settings Template

Create .claude/settings.json:

json 
{
  "mcpServers": {
    "fluent-mcp": {
      "command": "npx",
      "args": ["-y", "@modesty/fluent-mcp"],
      "env": {
        "SN_INSTANCE_URL": "https://your-instance.service-now.com",
        "SN_AUTH_TYPE": "basic",
        "SN_USER_NAME": "your-username",
        "SN_PASSWORD": "your-password"
      }
    }
  }
}
Credentials This template uses placeholder values and is safe to commit. For real credentials, save to .claude/settings.local.json (gitignored). Never commit real credentials.
5

Development Loop

The core Fluent workflow: download from instance → edit DSL locally → build → install back.

5.1 — Download Before You Start

bash 
now-sdk download src/

5.2 — Edit Files Locally

Open files in src/ using VS Code or Claude Code. You'll be writing Fluent DSL:

typescript 
// Example: defining a Script Include in Fluent
export const IncidentUtils = scriptInclude({
  name: 'IncidentUtils',
  script: () => {
    // your script logic here
  }
});
Tip Ask Claude Code to help — e.g., "Add a utility function to IncidentUtils that formats an incident number as INC0001234."

5.3 — Build (Compile Only)

Compile and check for errors without deploying to the instance:

bash 
now-sdk build

5.4 — Install to Instance

bash 
now-sdk install

This compiles your Fluent DSL and installs the metadata to your PDI. Verify in Studio that your changes are there.

5.5 — Handling Conflicts

bash 
now-sdk download src/ # Get latest from instance first
# Review and merge any differences manually
now-sdk install       # Then install your changes
Rule of thumb Download first, always. Especially if others share the instance.

bash

6.3 — Push Branch and Open PR

bash 
git push -u origin feature/my-first-change
gh pr create \
  --title "Add incident utility functions" \
  --body "Adds formatIncidentNumber to IncidentUtils."

6.4 — Merge and Clean Up

bash 
gh pr merge --squash
git checkout main
git pull
git branch -d feature/my-first-change
7

MCP Setup — Claude ↔ ServiceNow

What is MCP? Model Context Protocol (MCP) lets Claude Code talk directly to your ServiceNow instance. With MCP configured, you can ask Claude to query tables, inspect app metadata, and interact with your instance without leaving your terminal.
Note There is no official ServiceNow MCP server. This guide uses @modesty/fluent-mcp — a community package that wraps the Fluent SDK.

7.1 — Install fluent-mcp

bash 
npx -y @modesty/fluent-mcp --help

7.2 — Configure Credentials (Local Only)

Edit .claude/settings.json, replacing placeholders with real values:

json 
{
  "mcpServers": {
    "fluent-mcp": {
      "command": "npx",
      "args": ["-y", "@modesty/fluent-mcp"],
      "env": {
        "SN_INSTANCE_URL": "https://dev12345.service-now.com",
        "SN_AUTH_TYPE": "basic",
        "SN_USER_NAME": "admin",
        "SN_PASSWORD": "your-actual-password"
      }
    }
  }
}
Critical This file with real credentials must NEVER be committed to git. Rename it to .claude/settings.local.json (gitignored) and keep the placeholder template as .claude/settings.json.

7.3 — Restart Claude Code

bash 
# Exit (Ctrl+C or /exit), then restart
claude

7.4 — Verify MCP Connection

claude code 
/mcp

You should see fluent-mcp listed as a connected server.

7.5 — Try It

  • "List the Script Includes in my ServiceNow scoped app"
  • "Show me the tables defined in scope x_snc_training"
  • "What app metadata is installed on my instance?"
